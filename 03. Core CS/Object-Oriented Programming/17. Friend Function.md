# 1. What is a Friend Function?

A Friend Function is a function that is not a member of a class but has access to the private and protected members of that class.

> "Friend Function = External Function + Access to Private Members"

---

# 2. Why Do We Need Friend Functions?

- Access private members without making them public.
    
- Perform operations involving multiple classes.
    
- Operator Overloading.
    
- Increase flexibility.
    

---

# 3. Syntax

```cpp
class ClassName {
    friend returnType functionName(parameters);
};
```

### Example

```cpp
class Demo {
    friend void display(Demo obj);
};
```

---

# 4. Basic Example

```cpp
#include <iostream>
using namespace std;

class Student {
private:
    int age = 22;

    friend void display(Student s);
};

void display(Student s) {
    cout << s.age;
}

int main() {
    Student s;

    display(s);
}
```

### Output

```text
22
```

---

# 6. Properties of Friend Functions

1. Not a member of the class.
    
2. Can access private members.
    
3. Declared using `friend`.
    
4. Called like normal functions.
    
5. No `this` pointer.
    

---

# 7. Friend Function vs Member Function

|Friend Function|Member Function|
|---|---|
|Outside class|Inside class|
|No `this` pointer|Has `this` pointer|
|Declared using `friend`|Normal declaration|
|Can access private data|Can access private data|

---

# 8. Multiple Friend Functions

```cpp
class Demo {
    friend void show();
    friend void display();
};
```

Multiple friend functions are allowed.

---

# 9. Friend Class

Entire classes can also be made friends.

```cpp
class A {
    friend class B;
};
```

This allows all members of `B` to access private members of `A`.

# 11. Rules

1. Must be declared inside the class.
    
2. Defined outside the class.
    
3. Not inherited.
    
4. Not in the class scope.
    
5. Can access private and protected members.
    

---

# 12. Friend Function and Multiple Classes

```cpp
class B;

class A {
    friend void show(A, B);
};

class B {
    friend void show(A, B);
};
```

```text
       A        B
        \      /
         \    /
          \  /
        Friend Function
```

This is a common use case.

---

# 13. Common Mistakes

### Mistake

```cpp
class Demo {
private:
    int x;
};

void show(Demo d) {
    cout << d.x;
}
```

> Error: `x` is private.

---

### Correct

```cpp
class Demo {
    friend void show(Demo);
};
```

# 14. Interview Questions

### What is a Friend Function?

- A non-member function with access to private members.
    

---

### Is it a member of the class?

- No.
    

---

### Does it have a `this` pointer?

- No.
    

---

### Can Friend Functions be inherited?

- No.
    

---

### Can classes be friends?

- Yes.
    

---

# 15. Advantages

- Access private data.
    
- Useful in Operator Overloading.
    
- Supports interaction between multiple classes.
    

---

# 16. Disadvantages

- Breaks Encapsulation.
    
- Reduces Data Hiding.
    
- Increases coupling.
    

> Interviewers often ask:
> 
> "Is Friend Function against OOP principles?"
> 
> Answer:
> 
> "Yes, it partially violates Encapsulation."
