# 1. What is Abstraction?

Abstraction is the process of hiding implementation details and showing only the essential features of an object.

> "Hide HOW, show WHAT."

Users know **what** an object does, but not **how** it does it.

---

# 2. Why Do We Need Abstraction?

- Reduces complexity.
    
- Improves security.
    
- Makes code easier to understand.
    
- Allows implementation changes without affecting users.
    
- Increases maintainability.

# 4. How is Abstraction Achieved in C++?

1. Abstract Classes
    
2. Access Specifiers (`private`, `protected`)
    
3. Header Files
    

> For placements, the most important answer is: **Abstract Classes**.

---

# 5. Syntax (Abstract Class)

```cpp
class Shape {
public:
    virtual void draw() = 0;
};
```

> `= 0` indicates a Pure Virtual Function.

---

# 6. Basic Example

```cpp
#include <iostream>
using namespace std;

class Car {
public:
    void start() {
        cout << "Car Started";
    }
};

int main() {
    Car c;

    c.start();
}
```

### User sees:

```text
Car Started
```

### User does NOT see:

```text
- Battery Check
- Fuel Check
- Ignition Process
```

This is abstraction.

---

# 7. Abstraction Using Abstract Class

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() = 0;
};

class Dog : public Animal {
public:
    void sound() {
        cout << "Bark";
    }
};

int main() {
    Dog d;
    d.sound();
}
```

# 9. Rules

1. Abstraction hides implementation details.
    
2. Achieved using Abstract Classes.
    
3. Uses Pure Virtual Functions.
    
4. Cannot instantiate Abstract Classes.
    
5. Derived classes must implement Pure Virtual Functions.
    

---

# 10. Abstraction vs Encapsulation

|Abstraction|Encapsulation|
|---|---|
|Hides implementation|Hides data|
|Focuses on "What"|Focuses on "How to protect"|
|Uses abstract classes|Uses access specifiers|
|Simplifies usage|Improves security|

---

# 13. Common Mistakes

### Mistake

```cpp
class Animal {
public:
    void sound() {}
};
```

This is NOT an abstract class.

---

### Correct

```cpp
class Animal {
public:
    virtual void sound() = 0;
};
```
