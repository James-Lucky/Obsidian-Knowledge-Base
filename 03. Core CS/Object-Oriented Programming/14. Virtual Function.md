# 1. What is a Virtual Function?

A Virtual Function is a member function declared in the base class using the `virtual` keyword that allows a derived class to override it.

It enables **Runtime Polymorphism** (Dynamic Binding).

> "Virtual Functions decide which function to call at runtime."

---

# 2. Why Do We Need Virtual Functions?

Without virtual functions:

- Base class pointer calls base class function.
    

With virtual functions:

- Base class pointer calls derived class function.
    

---

# 3. Syntax

```cpp
class Base {
public:
    virtual void show() {
    }
};
```

---

# 4. Basic Example

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() {
        cout << "Animal Sound\n";
    }
};

class Dog : public Animal {
public:
    void sound() override {
        cout << "Bark\n";
    }
};

int main() {
    Animal* ptr;

    Dog d;

    ptr = &d;

    ptr->sound();
}
```

### Output

```text
Bark
```

---

# 5. Without Virtual Function

```cpp
class Animal {
public:
    void sound() {
        cout << "Animal";
    }
};

class Dog : public Animal {
public:
    void sound() {
        cout << "Dog";
    }
};
```

```cpp
Animal* ptr = &d;

ptr->sound();
```

### Output

```text
Animal
```

Reason:

```text
Compiler performs Early Binding.
```

---

# 6. With Virtual Function

```cpp
virtual void sound();
```

### Output

```text
Dog
```

Reason:

```text
Runtime performs Dynamic Binding.
```

---

# 7. How Virtual Functions Work

```text
Animal* ptr
      |
      v
     Dog

ptr->sound()
      |
      v
Runtime Checks Actual Object
      |
      v
Calls Dog::sound()
```

# 8. VTABLE and VPTR

Virtual Functions internally use:

1. VTABLE (Virtual Table)
    
2. VPTR (Virtual Pointer)
    

### VTABLE

- A table containing addresses of virtual functions.
    

### VPTR

- A hidden pointer added to each object.

# 9. Rules

1. Must be declared in the base class.
    
2. Can be overridden in the derived class.
    
3. Supports Runtime Polymorphism.
    
4. Cannot be static.
    
5. Constructors cannot be virtual.
    
6. Destructors can be virtual.
    

---

# 10. Pure Virtual Function

```cpp
virtual void show() = 0;
```

This makes the class abstract.

---

# 11. Virtual Function vs Normal Function

|Virtual Function|Normal Function|
|---|---|
|Runtime Binding|Compile-Time Binding|
|Uses VTABLE|No VTABLE|
|Supports Polymorphism|No Polymorphism|
|Slightly Slower|Faster|

---

# 12. Common Mistakes

### Mistake

```cpp
class Base {
public:
    static virtual void show();
};
```

> Error: Static functions cannot be virtual.

---

### Mistake

```cpp
class Demo {
public:
    virtual Demo();
};
```

> Error: Constructors cannot be virtual.

---

# 13. Constructor vs Destructor

|Constructor|Destructor|
|---|---|
|Cannot be Virtual|Can be Virtual|
|Object not fully created|Object already exists|

# 14. Interview Questions

### What is a Virtual Function?

- A function that enables Runtime Polymorphism.
    

---

### Why use Virtual Functions?

- To call the appropriate derived class function at runtime.
    

---

### What is VTABLE?

- A table storing addresses of virtual functions.
    

---

### What is VPTR?

- Hidden pointer pointing to the VTABLE.
    

---

### Can Constructors be Virtual?

- No.
    

---

### Can Destructors be Virtual?

- Yes.
    

---

### Which binding is used?

- Dynamic Binding.
