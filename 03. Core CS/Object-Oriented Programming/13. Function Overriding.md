# 1. What is Function Overriding?

Function Overriding occurs when a derived class provides its own implementation of a function that already exists in the base class.

> "Same function name + Same parameters + Inheritance"

The derived class overrides the behavior of the base class.

---

# 2. Why Do We Need Function Overriding?

- Achieve Runtime Polymorphism.
    
- Customize inherited behavior.
    
- Increase flexibility.
    
- Support Dynamic Binding.
    

---

# 3. Requirements for Function Overriding

1. Inheritance is required.
    
2. Function names must be the same.
    
3. Parameter lists must be the same.
    
4. Preferably use the `virtual` keyword in the base class.
    
5. Return types should be compatible.
    

---

# 4. Syntax

```cpp
class Base {
public:
    virtual void show() {
    }
};

class Derived : public Base {
public:
    void show() {
    }
};
```

---

# 5. Basic Example

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() {
        cout << "Animal Sound\n";
    }
};

class Dog : public Animal {
public:
    void sound() {
        cout << "Bark\n";
    }
};

int main() {
    Dog d;

    d.sound();
}
```

### Output

```text
Bark
```

# 7. Example with Base Pointer

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() {
        cout << "Animal\n";
    }
};

class Dog : public Animal {
public:
    void sound() {
        cout << "Dog\n";
    }
};

int main() {
    Animal* ptr;

    Dog d;

    ptr = &d;

    ptr->sound();
}
```

### Output

```text
Dog
```

> This is Runtime Polymorphism.

# 8. Function Overloading vs Function Overriding

|Function Overloading|Function Overriding|
|---|---|
|Same Class|Different Classes|
|No Inheritance|Requires Inheritance|
|Compile-Time|Run-Time|
|Different Parameters|Same Parameters|

---

# 9. Overriding Rules

|Rule|Required|
|---|---|
|Same Function Name|Yes|
|Same Parameters|Yes|
|Inheritance|Yes|
|Virtual Keyword|Recommended|
|Same Return Type|Preferred|

# 11. `override` Keyword (C++11)

Modern C++ provides the `override` keyword.

```cpp
class Dog : public Animal {
public:
    void sound() override {
        cout << "Bark";
    }
};
```

### Benefits

- Improves readability.
    
- Compiler checks correctness.
    
- Prevents accidental mistakes.
    

> Always prefer `override` in modern C++.

---

# 12. Common Mistakes

### Mistake 1

```cpp
class Dog : public Animal {
public:
    void sound(int x) {
    }
};
```

This is NOT overriding.

It is a completely different function.

---

### Mistake 2

```cpp
class Animal {
public:
    void sound() {}
};
```

Without `virtual`, runtime polymorphism will not occur.

# 13. Interview Questions

### What is Function Overriding?

- Redefining a base class function in the derived class.
    

---

### What are its requirements?

1. Inheritance
    
2. Same Name
    
3. Same Parameters
    

---

### Is Function Overriding Compile-Time or Run-Time?

- Run-Time Polymorphism.
    

---

### Is the `virtual` keyword mandatory?

- No, but required for runtime dispatch.
    

---

### What keyword should be used in modern C++?

```cpp
override
```
