# 1. What is Encapsulation?

Encapsulation is the process of binding data (variables) and methods (functions) together into a single unit (class) and restricting direct access to some of the object's data.

> "Data Hiding + Wrapping = Encapsulation"

---

# 2. Why Do We Need Encapsulation?

- Protects data from unauthorized access.
    
- Improves security.
    
- Increases maintainability.
    
- Makes code modular.
    
- Provides controlled access to data.
    

---

# 3. Real-Life Example

```text
ATM Machine
     |
     v
You can:
- Withdraw Money
- Check Balance

You CANNOT:
- Access bank database directly
```

Similarly, users should not directly access an object's private data.

---

# 4. Syntax

```cpp
class ClassName {
private:
    int data;

public:
    void setData(int value);
    int getData();
};
```

---

# 5. Basic Example

```cpp
#include <iostream>
using namespace std;

class Student {
private:
    int age;

public:
    void setAge(int a) {
        age = a;
    }

    int getAge() {
        return age;
    }
};

int main() {
    Student s1;

    s1.setAge(22);

    cout << s1.getAge();
}
```

### Output

```text
22
```

# 7. Components of Encapsulation

1. Data Members
    
2. Member Functions
    
3. Access Specifiers
    

---

## Access Specifiers

```cpp
public
private
protected
```

### Public

- Accessible everywhere.
    

### Private

- Accessible only inside the class.
    

### Protected

- Accessible in derived classes.
    

---

# 8. Rules

1. Data should generally be private.
    
2. Access should be provided through public methods.
    
3. Getter and Setter methods are commonly used.
    
4. Encapsulation is implemented using classes.
    

---

# 9. Getters and Setters

### Getter

```cpp
int getAge() {
    return age;
}
```

### Setter

```cpp
void setAge(int a) {
    age = a;
}
```

---

# 10. Advantages

- Data Hiding
    
- Security
    
- Flexibility
    
- Easy Maintenance
    
- Better Code Reusability
    

---

# 11. Disadvantages

- Slightly more code.
    
- Additional function calls.
    
- Can increase complexity for very small programs.

# 13. Encapsulation vs Abstraction

|Encapsulation|Abstraction|
|---|---|
|Hides data|Hides implementation|
|Achieved using access modifiers|Achieved using abstract classes|
|Focuses on security|Focuses on simplicity|

> This question is extremely common in interviews.

---

# 14. Common Mistakes

### Wrong

```cpp
class Student {
public:
    int age;
};
```

Everything is public.

---

### Better

```cpp
class Student {
private:
    int age;
};
```
