# 1. What is a Virtual Destructor?

A Virtual Destructor is a destructor declared with the `virtual` keyword in the base class.

It ensures that when a base class pointer points to a derived class object and is deleted, both the derived class and base class destructors are called in the correct order.

> "Always make the base class destructor virtual when using inheritance polymorphically."

---

# 2. Why Do We Need Virtual Destructors?

Without a virtual destructor:

- Only the base class destructor is called.
    
- Derived class resources are not released.
    
- Causes memory/resource leaks.
    

With a virtual destructor:

- Derived destructor is called first.
    
- Base destructor is called second.
    
- Proper cleanup occurs.
    

---

# 3. Syntax

```cpp
class Base {
public:
    virtual ~Base() {
    }
};
```

---

# 4. Basic Example

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual ~Base() {
        cout << "Base Destructor\n";
    }
};

class Derived : public Base {
public:
    ~Derived() {
        cout << "Derived Destructor\n";
    }
};

int main() {
    Base* ptr = new Derived();

    delete ptr;
}
```

### Output

```text
Derived Destructor
Base Destructor
```

# 5. Without Virtual Destructor

```cpp
class Base {
public:
    ~Base() {
        cout << "Base Destructor\n";
    }
};

class Derived : public Base {
public:
    ~Derived() {
        cout << "Derived Destructor\n";
    }
};
```

```cpp
Base* ptr = new Derived();

delete ptr;
```

### Output

```text
Base Destructor
```

### Problem

```text
Derived Destructor NOT Called
        |
        v
Resource Leak
```

# 9. Rules

1. Constructors cannot be virtual.
    
2. Destructors can be virtual.
    
3. Virtual destructors are inherited.
    
4. Base classes used polymorphically should always have virtual destructors.
    
5. Destruction occurs in reverse order of construction.
    

---

# 10. Virtual Destructor vs Normal Destructor

|Virtual Destructor|Normal Destructor|
|---|---|
|Supports polymorphism|No polymorphism|
|Proper cleanup|May leak resources|
|Uses VTABLE|No VTABLE|
|Recommended for inheritance|Fine for standalone classes|

# 12. Common Mistakes

### Mistake

```cpp
class Base {
public:
    ~Base() {}
};
```

Using this in polymorphic inheritance can be dangerous.

---

### Correct

```cpp
class Base {
public:
    virtual ~Base() {}
};
```

# 13. Interview Questions

### What is a Virtual Destructor?

- A destructor declared using the `virtual` keyword.
    

---

### Why do we use it?

- To ensure proper cleanup in inheritance.
    

---

### Can constructors be virtual?

- No.
    

---

### Can destructors be virtual?

- Yes.
    

---

### What happens if the base destructor is not virtual?

- Derived destructor may not execute.
    

---

### Is Virtual Destructor important?

- Extremely important for polymorphic classes.
