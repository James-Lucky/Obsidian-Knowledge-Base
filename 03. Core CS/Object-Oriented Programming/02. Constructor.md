# 1. What is Constructor?

A Constructor is a special member function of a class that is automatically called when an object is created.

Its main purpose is to initialize the object's data members.

> "Constructor initializes the object."

---

# 2. Why Do We Need Constructors?

Without constructors:

- Variables may contain garbage values.
    
- Initialization has to be done manually.
    
- Code becomes repetitive.
    

With constructors:

- Automatic initialization.
    
- Cleaner code.
    
- Better readability.
    
- Improves object management.
    

---

# 3. Properties of Constructor

- Same name as the class.
    
- No return type (not even `void`).
    
- Called automatically.
    
- Executes only once per object.
    
- Can be overloaded.
    
- Can have default arguments.
    

---

# 4. Syntax

```cpp
class ClassName {
public:
    ClassName() {
        // Initialization
    }
};
```

---

# 5. Basic Example

```cpp
#include <iostream>
using namespace std;

class Student {
public:
    string name;

    Student() {
        name = "Lucky";
    }
};

int main() {
    Student s1;

    cout << s1.name;
}
```

### Output

```text
Lucky
```

# 6. Types of Constructors

1. Default Constructor
    
2. Parameterized Constructor
    
3. Copy Constructor
    
4. Dynamic Constructor
    

---

## A. Default Constructor

A constructor with no parameters.

```cpp
class Demo {
public:
    Demo() {
        cout << "Constructor Called";
    }
};
```

---

## B. Parameterized Constructor

Accepts arguments.

```cpp
class Student {
public:
    string name;

    Student(string n) {
        name = n;
    }
};

int main() {
    Student s1("Lucky");

    cout << s1.name;
}
```

---

## C. Copy Constructor

Creates one object from another object.

```cpp
class Demo {
public:
    int x;

    Demo(int a) {
        x = a;
    }

    Demo(const Demo& obj) {
        x = obj.x;
    }
};
```

---

## D. Dynamic Constructor

Uses dynamic memory allocation.

```cpp
class Demo {
    int* ptr;

public:
    Demo() {
        ptr = new int(10);
    }
};
```

> Mostly asked in university exams; less common in placements.

---

# 7. Constructor Overloading

Multiple constructors in the same class with different parameters.

```cpp
class Student {
public:

    Student() {
        cout << "Default\n";
    }

    Student(int age) {
        cout << age;
    }
};
```

# 10. Rules of Constructor

1. Name must be same as class name.
    
2. Cannot have a return type.
    
3. Can be overloaded.
    
4. Cannot be inherited.
    
5. Can be public, private, or protected.
    
6. Called automatically.
    
7. Executes once per object.
    

---

# 11. Difference Between Function and Constructor

|Function|Constructor|
|---|---|
|Has return type|No return type|
|Called manually|Called automatically|
|Performs tasks|Initializes objects|
|Can have any name|Same as class name|

---

# 12. Common Mistakes

### Wrong

```cpp
class Demo {
public:
    void Demo() {}
};
```

This is NOT a constructor.

---

### Correct

```cpp
class Demo {
public:
    Demo() {}
};
```

# 13. Interview Questions

### What is a constructor?

- A special member function used to initialize objects.
    

### Can constructors be overloaded?

- Yes.
    

### Can constructors be virtual?

- No.
    

### Can constructors return values?

- No.
    

### Can constructors be inherited?

- No.
    

### How many times is a constructor called?

- Once per object creation.
