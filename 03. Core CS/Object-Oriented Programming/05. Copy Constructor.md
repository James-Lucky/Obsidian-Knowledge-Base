# 1. What is a Copy Constructor?

A Copy Constructor is a special constructor used to initialize one object using another object of the same class.

> "It creates a copy of an existing object."

---

# 2. Why Do We Need a Copy Constructor?

- Copy object data.
    
- Avoid shallow copy issues.
    
- Manage dynamic memory safely.
    
- Implement custom copy behavior.
    

### Example

```cpp
Student s1("Lucky", 22);
Student s2 = s1;
```

Here:

- `s1` → Existing object
    
- `s2` → New object (copy of `s1`)
    

---

# 3. Syntax

```cpp
ClassName(const ClassName& obj) {
    // Copy logic
}
```

### Example

```cpp
Demo(const Demo& obj) {
    x = obj.x;
}
```

---

# 4. Basic Example

```cpp
#include <iostream>
using namespace std;

class Student {
public:
    string name;

    Student(string n) {
        name = n;
    }

    Student(const Student& obj) {
        name = obj.name;
    }
};

int main() {
    Student s1("Lucky");
    Student s2 = s1;

    cout << s2.name;
}
```

### Output

```text
Lucky
```

# 5. How It Works

```text
Student s1("Lucky");
        |
        v
Student s2 = s1;
        |
        v
Copy Constructor Called
        |
        v
s2.name = s1.name
```

# 6. When is a Copy Constructor Called?

### Case 1: Object Initialization

```cpp
Demo d2 = d1;
```

---

### Case 2: Pass Object by Value

```cpp
void display(Demo obj);
```

---

### Case 3: Return Object by Value

```cpp
Demo getData();
```

> These three cases are frequently asked in interviews.

# 7. Default Copy Constructor

If you don't define a copy constructor, the compiler automatically provides one.

```cpp
class Demo {
public:
    int x;
};
```

Compiler-generated version:

```cpp
Demo(const Demo& obj) {
    x = obj.x;
}
```

---

# 8. Shallow Copy

A shallow copy copies memory addresses.

```cpp
class Demo {
public:
    int* ptr;

    Demo(int val) {
        ptr = new int(val);
    }
};
```

```text
Object 1
   |
   v
+-------+
| 1000  |
+-------+

Object 2
   |
   v
+-------+
| 1000  |
+-------+
```

Both objects point to the same memory.

---

# 9. Deep Copy

A deep copy creates separate memory.

```cpp
Demo(const Demo& obj) {
    ptr = new int(*obj.ptr);
}
```

```text
Object 1 ----> Memory A

Object 2 ----> Memory B
```

> Deep Copy prevents memory leaks and double deletion.

---

# 10. Complete Example

```cpp
#include <iostream>
using namespace std;

class Demo {
public:
    int* ptr;

    Demo(int x) {
        ptr = new int(x);
    }

    Demo(const Demo& obj) {
        ptr = new int(*obj.ptr);
    }

    ~Demo() {
        delete ptr;
    }
};

int main() {
    Demo d1(10);
    Demo d2 = d1;
}
```

# 12. Rules

1. Takes object reference as parameter.
    
2. Usually uses `const`.
    
3. Name must match class name.
    
4. No return type.
    
5. Called automatically when needed.
    

---

# 13. Difference Table

|Constructor|Copy Constructor|
|---|---|
|Creates new object|Copies existing object|
|Takes normal arguments|Takes object reference|
|Initializes values|Duplicates values|

---

## Shallow vs Deep Copy

|Shallow Copy|Deep Copy|
|---|---|
|Copies address|Copies data|
|Shared memory|Separate memory|
|Unsafe|Safe|
|Default behavior|Custom implementation|

---

# 14. Common Mistakes

### Wrong

```cpp
Demo(Demo obj);
```

This causes infinite recursive calls.

---

### Correct

```cpp
Demo(const Demo& obj);
```

---

# 15. Interview Questions

### What is a Copy Constructor?

- A constructor that initializes one object from another object.
    

---

### When is it called?

1. Object initialization.
    
2. Pass by value.
    
3. Return by value.
    

---

### What is the difference between shallow and deep copy?

- Shallow: Copies addresses.
    
- Deep: Copies actual data.
    

---

### Can a Copy Constructor be private?

- Yes.
    

---

### Why use `const`?

- Prevents accidental modification.
