# 1. What is the Diamond Problem?

The Diamond Problem occurs in Multiple Inheritance when a class inherits from two classes that both inherit from the same base class.

As a result, the derived class gets multiple copies of the base class members, causing ambiguity.

> "One base class inherited multiple times creates ambiguity."

---

# 2. Why is it Called the Diamond Problem?

The inheritance structure looks like a diamond.

```text
         A
        / \
       /   \
      B     C
       \   /
        \ /
         D
```

- `B` inherits from `A`
    
- `C` inherits from `A`
    
- `D` inherits from both `B` and `C`
    

---

# 3. Problem Statement

Which copy of `A` should `D` use?

```text
         A
        / \
       /   \
      B     C
       \   /
        \ /
         D
          |
          v
      Ambiguity!
```

---

# 4. Example

```cpp
#include <iostream>
using namespace std;

class A {
public:
    int x = 10;
};

class B : public A {

};

class C : public A {

};

class D : public B, public C {

};

int main() {
    D obj;

    cout << obj.x;
}
```

### Error

```text
Ambiguous Reference to x
```

---

# 5. Why Does This Happen?

```text
D
|
|----> B ----> A
|
|----> C ----> A
```

`D` contains:

```text
A (from B)
A (from C)
```

Two copies of `A` exist.

---

# 6. Memory Representation

```text
          D
       --------
       |      |
       v      v
       B      C
       |      |
       v      v
       A      A
```

This duplication causes ambiguity.

---

# 7. Solution: Virtual Inheritance

Use the `virtual` keyword.

```cpp
class B : virtual public A {

};

class C : virtual public A {

};
```

---

# 8. Complete Example

```cpp
#include <iostream>
using namespace std;

class A {
public:
    int x = 10;
};

class B : virtual public A {

};

class C : virtual public A {

};

class D : public B, public C {

};

int main() {
    D obj;

    cout << obj.x;
}
```

### Output

```text
10
```

---

# 9. How Virtual Inheritance Works

```text
         A
        / \
       /   \
      B     C
       \   /
        \ /
         D

Virtual Inheritance

          |
          v

Only ONE Copy of A
```

# 11. Diamond Problem vs Multiple Inheritance

|Multiple Inheritance|Diamond Problem|
|---|---|
|Inherits multiple classes|Ambiguity due to repeated inheritance|
|Valid concept|Side effect of multiple inheritance|
|Not always problematic|Causes compilation issues|

---

# 12. Rules

1. Diamond Problem occurs only in Multiple Inheritance.
    
2. Caused by duplicate base class instances.
    
3. Solved using Virtual Inheritance.
    
4. Creates ambiguity during member access.
    

---

# 13. Common Mistakes

### Mistake

```cpp
class B : public A {};
class C : public A {};
```

Without:

```cpp
virtual
```

ambiguity occurs.

---

### Correct

```cpp
class B : virtual public A {};
class C : virtual public A {};
```

# 14. Interview Questions

### What is the Diamond Problem?

- Ambiguity caused by multiple inheritance.
    

---

### Why does it occur?

- Multiple copies of the same base class.
    

---

### How is it solved?

```cpp
virtual
```

---

### Which inheritance type causes it?

- Multiple Inheritance.
    

---

### What is Virtual Inheritance?

- A mechanism to ensure only one copy of the base class exists.
