# 1. What is Constructor Overloading?

Constructor Overloading means having multiple constructors in the same class with different parameter lists.

The compiler determines which constructor to call based on the number, type, and order of arguments.

> "Same constructor name, different parameters."

---

# 2. Why Do We Need Constructor Overloading?

- Initialize objects in different ways.
    
- Improves flexibility.
    
- Makes code more readable.
    
- Avoids writing separate initialization functions.
    

### Example

```cpp
Student s1;
Student s2("Lucky");
Student s3("Lucky", 22);
```

Each object uses a different constructor.

---

# 3. Syntax

```cpp
class ClassName {
public:
    ClassName();
    ClassName(int x);
    ClassName(int x, int y);
};
```

---

# 4. Basic Example

```cpp
#include <iostream>
using namespace std;

class Student {
public:
    string name;
    int age;

    Student() {
        name = "Unknown";
        age = 0;
    }

    Student(string n) {
        name = n;
        age = 0;
    }

    Student(string n, int a) {
        name = n;
        age = a;
    }
};

int main() {
    Student s1;
    Student s2("Lucky");
    Student s3("Lucky", 22);

    cout << s1.name << endl;
    cout << s2.name << endl;
    cout << s3.name << " " << s3.age;
}
```

---

# 5. Constructor Selection

```text
Student s1;
        |
        v
Student()

---------------------

Student s2("Lucky");
        |
        v
Student(string)

---------------------

Student s3("Lucky",22);
        |
        v
Student(string,int)
```

---

# 6. Rules

1. Constructors must have the same name as the class.
    
2. Parameter list must be different.
    
3. Return type is not allowed.
    
4. Compiler performs overload resolution.
    
5. Number, type, or order of parameters should differ.
    

---

# 7. Invalid Example

### Wrong

```cpp
class Demo {
public:
    Demo();
    Demo();
};
```

> Error: Same signature.

---

### Correct

```cpp
class Demo {
public:
    Demo();
    Demo(int x);
};
```

---

# 8. Types of Overloading

### A. Different Number of Parameters

```cpp
Demo();
Demo(int x);
Demo(int x, int y);
```

---

### B. Different Data Types

```cpp
Demo(int x);
Demo(float y);
```

---

### C. Different Order

```cpp
Demo(int x, float y);
Demo(float y, int x);
```

# 10. Difference: Constructor vs Constructor Overloading

|Constructor|Constructor Overloading|
|---|---|
|Single constructor|Multiple constructors|
|One initialization method|Multiple initialization methods|
|Less flexible|More flexible|

---

# 11. Real-Life Example

```text
Car()

Creates default car.

Car("BMW")

Creates BMW.

Car("BMW",2026)

Creates BMW with model year.
```

---

# 12. Common Mistakes

### Mistake 1

```cpp
Demo(int);
Demo(int);
```

Same signature → Compilation Error.

---

### Mistake 2

```cpp
Demo(int,float);
Demo(float,int);
```

Valid because order is different.

---

# 13. Interview Questions

### What is constructor overloading?

- Multiple constructors with different parameter lists.
    

---

### Is constructor overloading an example of polymorphism?

- Yes.
    
- It is Compile-Time (Static) Polymorphism.
    

---

### Can constructors have default arguments?

- Yes.
    

```cpp
Demo(int x = 10);
```

---

### Can constructors be overloaded infinitely?

- Yes, as long as signatures are unique.
    

---

# 14. Placement Notes

- Frequently asked in interviews.
    
- Considered Compile-Time Polymorphism.
    
- Signature = Name + Parameters.
    
- Return type is not considered in overloading.
