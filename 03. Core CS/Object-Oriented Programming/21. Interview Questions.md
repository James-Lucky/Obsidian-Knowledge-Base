# One-Line Definitions

|Topic|Definition|
|---|---|
|Class|Blueprint of objects|
|Object|Instance of a class|
|Constructor|Initializes objects|
|Destructor|Cleans objects|
|Encapsulation|Data Hiding|
|Abstraction|Hide implementation details|
|Inheritance|Acquire properties of another class|
|Polymorphism|One entity, many forms|
|Virtual Function|Supports Runtime Polymorphism|
|Friend Function|Non-member with private access|
|Copy Constructor|Copies objects|
|Abstract Class|Class with Pure Virtual Function|

# Placement Cheat Sheet

```text
Constructor        -> Initializes Object
Destructor         -> Cleans Object
Copy Constructor   -> Copies Object

Encapsulation      -> Data Hiding
Abstraction        -> Hide Implementation
Inheritance        -> Code Reusability
Polymorphism       -> Many Forms

Compile-Time       -> Overloading
Run-Time           -> Overriding

Virtual Function   -> Dynamic Binding
Virtual Destructor -> Proper Cleanup

Diamond Problem    -> Multiple Inheritance
Solution           -> Virtual Inheritance

this Pointer       -> Current Object
Friend Function    -> Access Private Data

Shallow Copy       -> Copies Address
Deep Copy          -> Copies Data
```

# Last Minute Revision (5 Minutes)

```text
1. Four Pillars:
   - Encapsulation
   - Abstraction
   - Inheritance
   - Polymorphism

2. Constructor != Destructor

3. Constructor:
   - No Return Type
   - Automatic
   - Can Overload

4. Destructor:
   - Uses ~
   - Cannot Overload

5. Runtime Polymorphism:
   - Virtual Function
   - Function Overriding

6. Diamond Problem:
   - Multiple Inheritance
   - Solved by Virtual Inheritance

7. Deep Copy > Shallow Copy

8. Static Functions:
   - No this Pointer

9. Friend Functions:
   - Break Encapsulation

10. Most Asked:
   - Virtual Function
   - Copy Constructor
   - Destructor
   - Diamond Problem
   - VTABLE/VPTR
```
