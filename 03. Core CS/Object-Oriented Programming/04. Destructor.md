# 1. What is a Destructor?

A Destructor is a special member function that is automatically called when an object goes out of scope or is destroyed.

Its primary purpose is to release resources (memory, files, network connections, etc.) allocated by the object.

> "Constructor creates/initializes an object, Destructor destroys/cleans it up."

---

# 2. Why Do We Need Destructors?

- Free dynamically allocated memory.
    
- Prevent memory leaks.
    
- Close files.
    
- Release resources.
    
- Perform cleanup operations.
    

---

# 3. Syntax

```cpp
class ClassName {
public:
    ~ClassName() {
        // Cleanup code
    }
};
```

> A destructor is identified by the `~` (tilde) symbol before the class name.

---

# 4. Basic Example

```cpp
#include <iostream>
using namespace std;

class Demo {
public:
    Demo() {
        cout << "Constructor Called\n";
    }

    ~Demo() {
        cout << "Destructor Called\n";
    }
};

int main() {
    Demo obj;
}
```

### Output

```text
Constructor Called
Destructor Called
```

# 5. Constructor vs Destructor

```text

      |
      v

      |
      v
Program Executes
      |
      v
Destructor Called
      |
      v
Memory Released
```

---

# 6. Rules of Destructor

1. Name must be same as the class name prefixed with `~`.
    
2. Cannot have a return type.
    
3. Cannot take parameters.
    
4. Only one destructor is allowed per class.
    
5. Cannot be overloaded.
    
6. Called automatically.
    
7. Executes in reverse order of object creation.
    

---

# 7. Important Points

### Constructor

```cpp
Demo();
```

### Destructor

```cpp
~Demo();
```

---

# 8. Example with Dynamic Memory

```cpp
#include <iostream>
using namespace std;

class Demo {
    int* ptr;

public:
    Demo() {
        ptr = new int(10);
    }

    ~Demo() {
        delete ptr;
        cout << "Memory Freed";
    }
};

int main() {
    Demo obj;
}
```

### Why?

Without `delete`, memory allocated using `new` remains occupied.

This leads to:

```text
Memory Leak
      ↓
High RAM Usage
      ↓
Poor Performance
```

# 9. Order of Destructor Calls

```cpp
#include <iostream>
using namespace std;

class Demo {
public:
    Demo() {
        cout << "Created\n";
    }

    ~Demo() {
        cout << "Destroyed\n";
    }
};

int main() {
    Demo a;
    Demo b;
    Demo c;
}
```

### Output

```text
Created
Created
Created

Destroyed
Destroyed
Destroyed
```

### Execution Order

```text
Creation:
a -> b -> c

Destruction:
c -> b -> a
```

> Destructors follow LIFO (Last In, First Out).

# 11. Difference Table

|Constructor|Destructor|
|---|---|
|Initializes object|Cleans object|
|No `~` symbol|Uses `~` symbol|
|Can be overloaded|Cannot be overloaded|
|Can have parameters|Cannot have parameters|
|Multiple allowed|Only one allowed|

---

# 12. Common Mistakes

### Wrong

```cpp
~Demo(int x);
```

> Error: Destructor cannot have parameters.

---

### Wrong

```cpp
int ~Demo();
```

> Error: Destructor cannot return a value.

---

### Correct

```cpp
~Demo() {
    // Cleanup
}
```

---

# 13. Virtual Destructor

A destructor can be declared `virtual`.

```cpp
class Base {
public:
    virtual ~Base() {}
};
```

> Virtual Destructors are extremely important in inheritance and are frequently asked in interviews.

### What is a destructor?

---

- No.
    

---

- No.

### Can destructors be virtual?

- Yes.
