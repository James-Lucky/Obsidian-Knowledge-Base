### 22. Namespaces

Namespaces are used to avoid naming conflicts in large programs. They allow you to group related classes, functions, and variables under a unique name. For example, the C++ Standard Library uses the `std` namespace (`std::cout`, `std::vector`). They become important when multiple libraries have functions with the same name.

**Uses:**

- Avoid naming conflicts.
    
- Organize code.
    
- Separate libraries/modules.
    

---

### 23. Exception Handling

Exception Handling is a mechanism for handling runtime errors gracefully using `try`, `catch`, and `throw`. Instead of crashing the program, exceptions allow you to detect errors and recover from them.

**Uses:**

- Handle runtime errors.
    
- Prevent program crashes.
    
- Improve reliability.
    
- Error reporting.
    

---

### 24. File Handling

File Handling allows programs to read from and write to files using `ifstream`, `ofstream`, and `fstream`. It is useful for storing data permanently instead of keeping it only in memory.

**Uses:**

- Read files.
    
- Write files.
    
- Store application data.
    
- Logging and reports.
    

---

### 25. Templates

Templates enable Generic Programming. They allow you to write code once and use it with different data types (int, float, string, etc.).

**Uses:**

- Generic programming.
    
- Reusable code.
    
- Type independence.
    
- Basis of STL.
    

---

### 26. STL (Standard Template Library)

STL is a collection of ready-made containers, algorithms, and iterators provided by C++. It significantly reduces development time.

**Major Components:**

- Containers (`vector`, `map`, `set`)
    
- Algorithms (`sort`, `find`)
    
- Iterators
    

**Uses:**

- Faster development.
    
- Efficient data structures.
    
- Common algorithms.
    

---

### 27. Smart Pointers

Smart Pointers automatically manage memory and prevent memory leaks. Modern C++ prefers Smart Pointers over raw pointers.

**Types:**

- `unique_ptr`
    
- `shared_ptr`
    
- `weak_ptr`
    

**Uses:**

- Automatic memory management.
    
- Prevent memory leaks.
    
- Safer code.
    

---

### 28. RAII (Resource Acquisition Is Initialization)

RAII is a programming principle where resources (memory, files, locks) are acquired in constructors and released in destructors.

**Uses:**

- Automatic cleanup.
    
- Better resource management.
    
- Foundation of Smart Pointers.
    

---

### 29. Move Semantics

Move Semantics improve performance by transferring ownership of resources instead of copying them. Introduced in C++11.

**Uses:**

- Faster execution.
    
- Reduced copying.
    
- Efficient object transfers.
    

---

### 30. Lambda Functions

Lambda Functions are anonymous functions that can be written inline.

Example:

```cpp
[](int a, int b) {
    return a + b;
};
```

**Uses:**

- Short functions.
    
- STL algorithms.
    
- Event handling.
    
- Functional programming.
    

---

### 31. Rule of 3/5/0

These are guidelines for managing resources in C++.

- Rule of 3:
    
    - Destructor
        
    - Copy Constructor
        
    - Copy Assignment Operator
        
- Rule of 5:
    
    - Adds:
        
        - Move Constructor
            
        - Move Assignment Operator
            
- Rule of 0:
    
    - Prefer classes that don't manage resources directly.
        

**Uses:**

- Proper memory management.
    
- Prevent resource leaks.
    
- Modern C++ best practices.

### 32. Object Slicing

Object Slicing occurs when a derived object is assigned to a base object, causing the derived-specific data to be lost.

**Uses:**

- Understanding inheritance pitfalls.
    
- Writing safer inheritance code.

RTTI allows the program to determine the actual type of an object during execution.

**Features:**

**Uses:**

- Debugging.
    
- Polymorphic systems.

### 34. Dynamic Casting

**Uses:**
