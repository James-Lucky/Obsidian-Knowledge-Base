# 1. What is Inheritance?

Inheritance is an OOP mechanism in which one class acquires the properties and behaviors of another class.

- Existing Class → Parent/Base/Super Class
    
- New Class → Child/Derived/Sub Class
    

> "Inheritance promotes code reusability."

---

# 2. Why Do We Need Inheritance?

- Code Reusability
    
- Easy Maintenance
    
- Reduced Code Duplication
    
- Better Organization
    
- Extensibility
    

---

# 3. Real-Life Example

```text
          Animal
             |
    ------------------
    |                |
   Dog              Cat
```

Both `Dog` and `Cat` inherit common properties from `Animal`.

---

# 4. Syntax

```cpp
class ChildClass : accessSpecifier ParentClass {
    // Body
};
```

### Example

```cpp
class Dog : public Animal {

};
```

---

# 5. Basic Example

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    void eat() {
        cout << "Eating";
    }
};

class Dog : public Animal {

};

int main() {
    Dog d;

    d.eat();
}
```

### Output

```text
Eating
```

# 6. Types of Inheritance

1. Single Inheritance
    
2. Multiple Inheritance
    
3. Multilevel Inheritance
    
4. Hierarchical Inheritance
    
5. Hybrid Inheritance
    

---

# 7. Single Inheritance

One child inherits from one parent.

```text
Parent
   |
   v
Child
```

```cpp
class Animal {};

class Dog : public Animal {};
```

---

# 8. Multiple Inheritance

One child inherits from multiple parents.

```text
   Parent1      Parent2
       \         /
        \       /
         \     /
           Child
```

```cpp
class A {};
class B {};

class C : public A, public B {};
```

---

# 9. Multilevel Inheritance

Inheritance in multiple levels.

```text
Grandparent
      |
      v
Parent
      |
      v
Child
```

```cpp
class A {};
class B : public A {};
class C : public B {};
```

---

# 10. Hierarchical Inheritance

One parent, multiple children.

```text
          Parent
         /      \
        /        \
       v          v
    Child1     Child2
```

```cpp
class Animal {};

class Dog : public Animal {};
class Cat : public Animal {};
```

---

# 11. Hybrid Inheritance

Combination of multiple inheritance types.

```text
          A
         / \
        /   \
       B     C
        \   /
         \ /
          D
```

> Hybrid Inheritance often leads to the Diamond Problem.

---

# 12. Access Modes in Inheritance

### Public Inheritance

```cpp
class B : public A {};
```

### Protected Inheritance

```cpp
class B : protected A {};
```

### Private Inheritance

```cpp
class B : private A {};
```

---

# 13. Access Control Table

|Base Class|Public|Protected|Private|
|---|---|---|---|
|Public Inheritance|Public|Protected|Not Accessible|
|Protected Inheritance|Protected|Protected|Not Accessible|
|Private Inheritance|Private|Private|Not Accessible|

> This table is a favorite interview question.

# 14. Constructor and Destructor Order

```text
Creation:

Parent
   ↓
Child

----------------

Destruction:

Child
   ↓
Parent
```

### Example

```cpp
class A {
public:
    A() {
        cout << "A";
    }
};

class B : public A {
public:
    B() {
        cout << "B";
    }
};
```

### Output

```text
AB
```

# 16. Common Mistakes

### Mistake

```cpp
class Child : Parent {};
```

Works in some cases but explicitly writing:

```cpp
class Child : public Parent {};
```

is preferred.

---

### Mistake

Trying to access:

```cpp
private
```

members of the base class.

# 17. Interview Questions

### What is Inheritance?

- Acquiring properties of another class.
    

---

### Why use Inheritance?

---

### Name the types of Inheritance.

1. Single

3. Multilevel
    
4. Hierarchical

---

### Which inheritance causes the Diamond Problem?

- Multiple/Hybrid Inheritance.
    

---

### What is the order of constructor execution?

```text
Parent -> Child
```

---

### What is the order of destructor execution?

```text
Child -> Parent
```
