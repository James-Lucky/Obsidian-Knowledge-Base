# 1. What is an Abstract Class?

An Abstract Class is a class that contains at least one Pure Virtual Function.

A Pure Virtual Function is declared using `= 0`.

> "A class with one or more Pure Virtual Functions is called an Abstract Class."

---

# 2. Why Do We Need Abstract Classes?

- Achieve abstraction.
    
- Define a common interface for derived classes.
    
- Force derived classes to implement required methods.
    
- Improve code reusability and design.
    

---

# 3. Syntax

```cpp
class ClassName {
public:
    virtual void functionName() = 0;
};
```

---

# 4. Example

```cpp
class Animal {
public:
    virtual void sound() = 0;
};
```

Since `sound()` is a Pure Virtual Function, `Animal` becomes an Abstract Class.

---

# 5. Pure Virtual Function

### Syntax

```cpp
virtual returnType functionName() = 0;
```

### Example

```cpp
virtual void display() = 0;
```

---

# 6. Basic Program

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() = 0;
};

class Dog : public Animal {
public:
    void sound() {
        cout << "Bark";
    }
};

int main() {
    Dog d;
    d.sound();
}
```

### Output

```text
Bark
```

---

# 7. Important Rule

You cannot create objects of an Abstract Class.

### Invalid

```cpp
Animal a;
```

### Error

```text
Cannot instantiate abstract class.
```

---

# 8. Derived Class Rule

If a derived class does not implement all Pure Virtual Functions, it also becomes abstract.

```cpp
class Animal {
public:
    virtual void sound() = 0;
};

class Dog : public Animal {
};
```

Dog is also an Abstract Class.

# 10. Properties of Abstract Classes

1. Cannot create objects.
    
2. Can contain constructors.
    
3. Can contain destructors.
    
4. Can contain normal functions.
    
5. Must contain at least one Pure Virtual Function.
    

---

# 11. Constructor in Abstract Class

```cpp
class Animal {
public:
    Animal() {
        cout << "Constructor";
    }

    virtual void sound() = 0;
};
```

> Abstract Classes can have constructors.

---

# 12. Difference Table

|Abstract Class|Normal Class|
|---|---|
|Has Pure Virtual Function|No Pure Virtual Function|
|Cannot create objects|Can create objects|
|Used for abstraction|Used for implementation|
|Defines a contract|Provides complete behavior|

---

## Pure Virtual Function vs Virtual Function

|Pure Virtual Function|Virtual Function|
|---|---|
|`virtual void f()=0;`|`virtual void f();`|
|No implementation required|Can have implementation|
|Makes class abstract|Does not make class abstract|

# 14. Common Mistakes

### Wrong

```cpp
class Animal {
public:
    void sound() {}
};
```

Not an Abstract Class.

---

### Correct

```cpp
class Animal {
public:
    virtual void sound() = 0;
};
```

# 15. Interview Questions

### What is an Abstract Class?

- A class containing at least one Pure Virtual Function.
    

---

### Can we create objects of an Abstract Class?

- No.
    

---

### Can an Abstract Class have constructors?

- Yes.
    

---

### Can it have destructors?

- Yes.
    

---

### What is a Pure Virtual Function?

```cpp
virtual void show() = 0;
```

---

### Can an Abstract Class have normal methods?

- Yes.
