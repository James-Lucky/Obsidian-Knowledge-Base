# 1. What are Static Members?

Static Members belong to the class itself rather than to individual objects.

There are two types of static members:

1. Static Data Members
    
2. Static Member Functions
    

> "One copy shared by all objects."

---

# 2. Why Do We Need Static Members?

- Share common data among all objects.
    
- Save memory.
    
- Count objects.
    
- Store class-level information.
    

---

# 3. Static Data Member

A static data member is shared among all objects of a class.

### Syntax

```cpp
class ClassName {
public:
    static int count;
};
```

---

# 4. Declaration vs Definition

### Declaration (Inside Class)

```cpp
class Demo {
public:
    static int count;
};
```

### Definition (Outside Class)

```cpp
int Demo::count = 0;
```

> This is one of the most commonly forgotten interview points.

---

# 5. Basic Example

```cpp
#include <iostream>
using namespace std;

class Student {
public:
    static int count;

    Student() {
        count++;
    }
};

int Student::count = 0;

int main() {
    Student s1;
    Student s2;
    Student s3;

    cout << Student::count;
}
```

### Output

```text
3
```

---

# 6. Memory Representation

```text
        Student Class
        +-----------+
        | count = 3 |
        +-----------+
             ^
             |
   ---------------------
   |         |         |
   v         v         v
  s1        s2        s3
```

There is only ONE copy of `count`.

---

# 7. Static Member Function

Static member functions belong to the class.

### Syntax

```cpp
class Demo {
public:
    static void show();
};
```

---

# 8. Example

```cpp
#include <iostream>
using namespace std;

class Demo {
public:
    static void display() {
        cout << "Static Function";
    }
};

int main() {
    Demo::display();
}
```

### Output

```text
Static Function
```

---

# 9. Accessing Static Members

### Using Class Name

```cpp
Student::count;
```

### Using Object

```cpp
s1.count;
```

> Prefer using the class name.

---

# 10. Rules of Static Members

### Static Data Members

1. Shared among all objects.
    
2. Memory allocated only once.
    
3. Must be defined outside the class.
    
4. Accessible using class name.
    

---

### Static Member Functions

1. Can access only static members.
    
2. Do not have a `this` pointer.
    
3. Can be called without objects.
    
4. Belong to the class.
    

---

# 11. Why `this` is Not Available?

```cpp
class Demo {
public:
    static void show() {
        cout << this;
    }
};
```

### Error

```text
Static functions belong to the class,
not to an object.
```

No object means no `this` pointer.

# 13. Static vs Non-Static

|Static|Non-Static|
|---|---|
|Belongs to class|Belongs to object|
|One copy|Multiple copies|
|No `this`|Has `this`|
|Access via class|Access via object|

---

# 14. Real-Life Example

### Counting Objects

```cpp
class Employee {
public:
    static int totalEmployees;
};
```

```text
Employee 1
Employee 2
Employee 3

Total = 3
```

# 15. Common Mistakes

### Mistake

```cpp
class Demo {
public:
    static int x = 10;
};
```

This is invalid before C++17 for non-const integral types.

---

### Correct

```cpp
class Demo {
public:
    static int x;
};

int Demo::x = 10;
```

---

### Mistake

```cpp
static void show() {
    cout << this;
}
```

> `this` is not available.

# 16. Interview Questions

### What is a Static Member?

- A member shared among all objects.
    

---

### How many copies exist?

- Only one.
    

---

### Can static functions access non-static variables?

- No.
    

---

### Can static functions use `this`?

- No.
    

---

### Where are static variables defined?

- Outside the class.
