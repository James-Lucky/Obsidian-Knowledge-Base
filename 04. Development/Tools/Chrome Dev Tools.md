| #   | Feature                                  | Description                                                                                                                                                                                                                                |
| --- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 1   | **CSS Shadow Editor**                    | Visual tool in the **Styles** panel to tweak `box-shadow` (position, blur, spread, inset) without external generators.                                                                                                                     |
| 2   | **Log Points & Conditional Breakpoints** | • **Log Points** – right‑click a line number in **Sources** → _Add log_ to output a message/variable without changing code (persists across reloads).• **Conditional Breakpoints** – pause only when a specified condition evaluates true. |
| 3   | **Code Coverage Tab**                    | Shows what percentage of JS and CSS is actually executed on initial load; helps locate dead code for lazy‑loading or removal.                                                                                                              |
| 4   | **“Break On” DOM Modifications**         | In **Elements**, right‑click → _Break on_ → _subtree modifications / attribute changes / node removal_ to pause exactly when the DOM is altered.                                                                                           |
| 5   | **Emulate a Focused Page**               | Found under **Rendering** → _Emulate focused page_; keeps menus/pop‑ups open while you interact with DevTools.                                                                                                                             |
| 6   | **Rendering Tab Features**               | • Highlights scrolling performance issues.• Simulates light/dark mode.• Emulates visual impairments (e.g., color‑blindness) for accessibility testing.                                                                                     |
| 7   | **CSS Overview**                         | Audits site styles: total elements, unused rules, color palette consistency, font‑size variance.                                                                                                                                           |
| 8   | **Advanced Screenshots**                 | • **Capture Node Screenshot** – saves an image of a selected element.• **Capture Full‑Size Screenshot** – captures the whole page, including off‑screen areas.                                                                             |
| 9   | **Copy Console to Clipboard**            | Right‑click the console → _Copy Console_ to grab the full log history (handy for pasting into AI tools).                                                                                                                                   |
| 10  | **Animations Tab**                       | Record, replay, scrub, and slow down CSS animations & transitions to fine‑tune timing.                                                                                                                                                     |
| 11  | **Network Throttling**                   | Simulate slow 3G, offline, or custom bandwidth conditions to test loading states and race conditions.                                                                                                                                      |
| 12  | **`debugger` Keyword**                   | Insert `debugger;` in JS code to create a hard‑coded breakpoint that pauses execution automatically.                                                                                                                                       |
| 13  | **Snippets (Custom Browser Scripts)**    | Save reusable JS snippets in **Sources** → _Snippets_ (e.g., list all page links) and run them on demand.                                                                                                                                  |
| 14  | **Cache Clearing via Reload Button**     | With DevTools open, long‑press the browser’s reload button to reveal **Empty Cache and Hard Reload**.                                                                                                                                      |
