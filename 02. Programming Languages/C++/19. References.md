# C++ 

## What is a Reference?

A reference is an alias (another name) for an existing variable.

Once a reference is created, it refers to the same memory location as the original variable.

---

## Syntax

```cpp
dataType& referenceName = variableName;
```

---

## Example

```cpp
int number = 10;
int& ref = number;

std::cout << ref;
```

### Output

```text
10
```

---

## Memory Representation

```text
number ---> 10
ref    ---> 10
```

Both `number` and `ref` refer to the same memory location.

---

## Modifying Through References

```cpp
int number = 10;
int& ref = number;

ref = 50;

std::cout << number;
```

### Output

```text
50
```

Changing the reference changes the original variable.

---

## Reference vs Variable

```cpp
int x = 5;
int& y = x;

y = 20;
```

Result:

```text
x = 20
y = 20
```

---

## Rules for References

1. Must be initialized during declaration.
    
2. Cannot refer to `nullptr`.
    
3. Cannot be changed to reference another variable.
    
4. Must refer to a valid object.
    

### Invalid Example

```cpp
int& ref; // Error
```

---

## References in Functions

### Pass by Value

```cpp
void increment(int x) {
    x++;
}
```

Changes do not affect the original variable.

---

### Pass by Reference

```cpp
void increment(int& x) {
    x++;
}
```

Changes affect the original variable.

---

### Example

```cpp
int num = 10;

increment(num);

std::cout << num;
```

### Output

```text
11
```

> [!important]  
> Pass by reference avoids unnecessary copying and improves performance.

---

## Constant References

```cpp
const int& ref = number;
```

- Cannot modify the value through the reference.
    
- Commonly used in function parameters.
    

---

## References with Structures

```cpp
struct Student {
    std::string name;
};

Student s{"Lucky"};

Student& ref = s;
```

---

## Lvalue and Rvalue References

### Lvalue Reference

```cpp
int x = 10;

int& ref = x;
```

### Rvalue Reference (C++11)

```cpp
int&& ref = 10;
```

Rvalue references are used in:

- Move Semantics
    
- Perfect Forwarding
    
- Performance Optimization
    

> [!note]  
> Rvalue references are an advanced topic and will be covered later.

---

## References vs Pointers

|References|Pointers|
|---|---|
|Cannot be null|Can be null|
|Must be initialized|Can be uninitialized|
|Cannot be reassigned|Can point elsewhere|
|Easier to use|More flexible|

---

## Important Points

- References are aliases for existing variables.
    
- They must be initialized.
    
- Changes through a reference affect the original variable.
    
- `const` references are widely used in modern C++.
    

---

## Common Mistakes

- Declaring references without initialization.
    
- Confusing references with pointers.
    
- Assuming references can be reassigned.
    
- Forgetting `const` in function parameters.
    

---

## Extra Information

### Address Comparison

```cpp
int x = 10;
int& ref = x;

std::cout << &x << "\n";
std::cout << &ref;
```

Both addresses will be the same because they refer to the same memory location.
