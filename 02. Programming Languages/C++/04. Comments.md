# C++ 

## What are Comments?

Comments are lines of text in a program that are ignored by the compiler. They are used to explain code, improve readability, and temporarily disable code during debugging.

---

## Types of Comments

C++ supports two types of comments:

1. Single-Line Comments
    
2. Multi-Line Comments
    

---

## Single-Line Comments

Single-line comments start with `//`.

```cpp
// This is a single-line comment

int age = 21;
```

Everything after `//` on the same line is ignored by the compiler.

---

## Multi-Line Comments

Multi-line comments start with `/*` and end with `*/`.

```cpp
/*
    This is a
    multi-line comment.
*/
```

---

## Example

```cpp
#include <iostream>

int main() {
    // Print a message
    std::cout << "Hello, World!";

    return 0;
}
```

---

## Why Use Comments?

- Explain complex logic.
    
- Improve code readability.
    
- Help other developers understand the code.
    
- Temporarily disable code for testing and debugging.
    
- Add notes and reminders.
    

---

## Commenting Out Code

```cpp
int x = 10;

// std::cout << x;
```

The commented code will not execute.

---

## Nested Comments

C++ does **not** support nested multi-line comments.

```cpp
/*
    Outer Comment

    /*
        Inner Comment
    */
*/
```

The above code will produce an error.

---

## Documentation Comments

Some tools (e.g., Doxygen) use comments to generate documentation.

```cpp
/**
 * Adds two numbers.
 */
int add(int a, int b) {
    return a + b;
}
```

---

## Important Points

- `//` is used for single-line comments.
    
- `/* */` is used for multi-line comments.
    
- Comments are ignored by the compiler.
    
- Avoid writing unnecessary comments for obvious code.
    

---

## Common Mistakes

- Forgetting to close `*/`.
    
- Writing nested multi-line comments.
    
- Over-commenting simple code.
    
- Using comments instead of meaningful variable names.
    

---

## Extra Information

### Good vs Bad Comments

#### Bad

```cpp
// Increment x by 1
x++;
```

#### Good

```cpp
// Calculate bonus points for completing the level.
bonusPoints++;
```

> [!tip]  
> Write comments to explain **why** the code exists, not just **what** the code does.
