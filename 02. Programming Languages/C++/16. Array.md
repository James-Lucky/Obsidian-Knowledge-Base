# C++ Arrays

## What is an Array?

An array is a collection of elements of the same data type stored in contiguous memory locations.

Instead of creating multiple variables, arrays allow us to store multiple values under a single name.

---

## Syntax

```cpp
dataType arrayName[size];
```

### Example

```cpp
int numbers[5];
```

This creates an array that can store 5 integers.

---

## Initializing Arrays

### Method 1

```cpp
int numbers[5] = {10, 20, 30, 40, 50};
```

### Method 2

```cpp
int numbers[] = {10, 20, 30};
```

The compiler automatically determines the size.

---

## Accessing Elements

Array indexing starts at `0`.

```cpp
int numbers[] = {10, 20, 30};

std::cout << numbers[0];
```

### Output

```text
10
```

---

## Modifying Elements

```cpp
int numbers[3] = {10, 20, 30};

numbers[1] = 100;
```

Result:

```cpp
{10, 100, 30}
```

---

## Array Size

Use `sizeof()` to determine the number of elements.

```cpp
int numbers[] = {10, 20, 30, 40};

int size = sizeof(numbers) / sizeof(numbers[0]);
```

### Output

```text
4
```

> [!important]  
> This method works only for arrays in the same scope, not when passed to functions.

---

## Looping Through Arrays

### Using `for`

```cpp
int numbers[] = {1, 2, 3, 4};

for (int i = 0; i < 4; i++) {
    std::cout << numbers[i] << " ";
}
```

---

### Using Range-Based `for`

```cpp
int numbers[] = {1, 2, 3, 4};

for (int num : numbers) {
    std::cout << num << " ";
}
```

---

## Multidimensional Arrays

### Two-Dimensional Array

```cpp
int matrix[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};
```

Accessing elements:

```cpp
std::cout << matrix[1][2];
```

### Output

```text
6
```

---

## Memory Representation

```text
Index:   0   1   2   3
Value:  10  20  30  40
```

Arrays store elements in contiguous memory locations.

---

## Important Points

- Arrays store elements of the same type.
    
- Indexing starts at `0`.
    
- Arrays have a fixed size.
    
- Elements are stored in contiguous memory.
    

---

## Common Mistakes

- Accessing an invalid index.
    

```cpp
int arr[3];

arr[5]; // Undefined Behavior
```

- Forgetting that indexing starts from `0`.
    
- Assuming arrays can change size dynamically.
    
- Using arrays when `std::vector` would be more appropriate.
    

---

## Extra Information

### Arrays vs Vectors

|Array|Vector|
|---|---|
|Fixed Size|Dynamic Size|
|Faster|Slightly Slower|
|Built-in|STL Container|

```cpp
std::vector<int> nums = {1, 2, 3};
```

> [!note]  
> In modern C++, `std::vector` is generally preferred over raw arrays because it is safer and resizable.
