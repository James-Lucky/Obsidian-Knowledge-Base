# 1. Introduction to Functions

A function is a reusable block of code designed to perform a specific task. Instead of writing the same code repeatedly, developers place that code inside a function and call it whenever needed.

Functions are one of the most fundamental concepts in programming because they promote modularity. Large programs are usually divided into smaller functions, where each function handles one responsibility.

Think of a function like a coffee machine:

- You provide input (coffee beans, water).
- The machine performs a task.
- It produces an output (coffee).

Similarly:

```
int add(int a, int b) {
    return a + b;
}
```

- Inputs: `a` and `b`
- Process: Addition
- Output: Sum

# 2. Function Parameters

Parameters are variables listed in the function definition. They act as placeholders for values that will be supplied when the function is called.

```
void greet(string name) {
    cout << "Hello " << name;
}
```

Calling:

```
greet("Lucky");
```

Output:

```
Hello Lucky
```

### Explanation

When the function is called:

```
greet("Lucky");
```

The value `"Lucky"` is copied into the parameter `name`.

Memory Flow:

```
"Lucky"
   ↓
name parameter
   ↓
cout
```

---

# 3. Parameters vs Arguments

Students often confuse these two terms.

|Term|Meaning|
|---|---|
|Parameter|Variable in function definition|
|Argument|Actual value passed to function|

Example:

```
void add(int x, int y)
```

Parameters:

```
x
y
```

Function call:

```
add(10,20);
```

Arguments:

```
10
20
```

Think:

> Parameter = Placeholder  
> Argument = Actual Value

# 4. Default Parameters

Default parameters provide predefined values.

```
void greet(string name="Guest") {
    cout<<name;
}
```

Usage:

```
greet();
greet("Lucky");
```

Output:

```
Guest
Lucky
```

### Why Use Them?

Imagine creating a game:

```
createPlayer("Unknown");
```

Instead:

```
void createPlayer(string name="Player1");
```

Now:

```
createPlayer();
```

automatically creates:

```
Player1
```

### Rules

Defaults must start from the right.

Correct:

```
void func(int x,int y=5);
```

Wrong:

```
void func(int x=5,int y);
```

# 5. Multiple Parameters

Functions can accept several values.

```
void student(string name,int age,float cgpa)
{
    cout<<name<<endl;
    cout<<age<<endl;
    cout<<cgpa;
}
```

Calling:

```
student("Lucky",22,8.9);
```

Output:

```
Lucky
22
8.9
```

Multiple parameters make functions more flexible.

# 6. Return Values

A return value sends data back to the caller.

```
int square(int n)
{
    return n*n;
}
```

Usage:

```
int result=square(5);
```

Output:

```
25
```

### Function Types

|Type|Example|
|---|---|
|No Parameter, No Return|`void hello()`|
|Parameter, No Return|`void print(int x)`|
|No Parameter, Return|`int getAge()`|
|Parameter, Return|`int add(int a,int b)`|

# 7. Pass by Value

By default, C++ copies variables.

```
void increment(int x)
{
    x++;
}
```

```
int num=10;
increment(num);

cout<<num;
```

Output:

```
10
```

### Explanation

```
num =10
      ↓ copy
x =10
x++
x=11

num remains 10
```

Advantages:

- Safe
- Original value unchanged

Disadvantages:

- Memory overhead for large objects

---

# 8. Pass by Reference

Pass by reference uses the original variable.

```
void increment(int& x)
{
    x++;
}
```

```
int num=10;

increment(num);

cout<<num;
```

Output:

```
11
```

Memory:

```
num ----+
        |
        +----> x
```

Both refer to the same memory location.

Use references when:

- Modifying variables.
- Avoiding expensive copies.
- Working with large objects.

# 9. Passing Arrays

Arrays are automatically passed as pointers.

```
void display(int arr[],int size)
{
    for(int i=0;i<size;i++)
    {
        cout<<arr[i]<<" ";
    }
}
```

```
int nums[]={1,2,3,4};

display(nums,4);
```

Output:

```
1 2 3 4
```

Internally:

```
int* arr
```

is used.

Best practice:

```
void display(const int arr[],int size);
```

# 10. Passing Structures

Structures can also be passed to functions.

```
struct Student
{
    string name;
    int age;
};
```

Function:

```
void show(Student s)
{
    cout<<s.name;
}
```

Better:

```
void show(const Student& s)
```

Reason:

- No copying.
- Faster.
- Cannot modify object.

Example:

```
Student s1={"Lucky",22};

show(s1);
```

# 11. Function Overloading

Multiple functions can have the same name if their parameters differ.

```
void print(int x);
void print(double x);
void print(string x);
```

```
print(10);
print(3.14);
print("Hello");
```

Compiler decides which function to call.

Advantages:

- Cleaner APIs
- Better readability

Example:

```
int add(int a,int b);

double add(double a,double b);
```

# 12. Scope

Scope determines where variables can be accessed.

### Local Scope

```
void test()
{
    int x=10;
}
```

`x` exists only inside `test()`.

### Global Scope

```
int x=100;
```

Accessible everywhere.

Example:

```
int score=100;

void display()
{
    cout<<score;
}
```

# 13. Recursion

Recursion occurs when a function calls itself.

```
int factorial(int n)
{
    if(n==1)
        return 1;

    return n*factorial(n-1);
}
```

```
factorial(5)
```

Execution:

```
5*4*3*2*1
```

Output:

```
120
```

Every recursive function needs:

1. Base case
2. Recursive case

Without a base case:

```
void infinite()
{
    infinite();
}
```

Result:

```
Stack Overflow
```

# 14. Lambda Functions

Introduced in C++11.

Anonymous functions:

```
auto add=[](int a,int b)
{
    return a+b;
};

cout<<add(10,20);
```

Output:

```
30
```

Syntax:

```
[capture](parameters)
{
    body
};
```

Example:

```
int x=10;

auto show=[x]()
{
    cout<<x;
};
```

Output:

```
10
```

Lambdas are heavily used in:

- STL
- Sorting
- Algorithms
- Multithreading

# 15. Real-Life Example

Bank Application:

```
double deposit(double balance,double amount)
{
    return balance+amount;
}

double withdraw(double balance,double amount)
{
    return balance-amount;
}

void display(double balance)
{
    cout<<"Balance: "<<balance;
}
```

Usage:

```
double balance=1000;

balance=deposit(balance,500);

display(balance);
```

Output:

```
Balance: 1500
```

This demonstrates how large applications consist of many small functions.

# 16. Best Practices

### Use Meaningful Names

Good:

```
calculateTax()
```

Bad:

```
abc()
```

---

### Keep Functions Small

Bad:

```
processEverything()
```

Good:

```
loginUser()
calculateTax()
sendEmail()
```

---

### Prefer Const References

```
void display(const string& name)
```

Benefits:

- Faster
- Safer
- No copies

# 17. Common Mistakes

### Missing Return

```
int add()
{
}
```

---

### Infinite Recursion

```
void f()
{
    f();
}
```

---

### Incorrect Parameters

```
add(10);
```

when:

```
add(int,int);
```

---

### Forgetting Function Declaration

```
main()
{
    greet();
}
```

without:

```
void greet();
```
