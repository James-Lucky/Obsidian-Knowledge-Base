# C++ Constants

## What are Constants?

Constants are variables whose values cannot be changed after they are initialized.

They are used when a value should remain fixed throughout the program.

---

## Using `const`

### Syntax

```cpp
const dataType variableName = value;
```

### Example

```cpp
const int DAYS_IN_WEEK = 7;
```

Attempting to modify a constant will result in an error.

```cpp
const int DAYS = 7;

DAYS = 10; // Error
```

---

## Why Use Constants?

- Prevent accidental modifications.
    
- Improve code readability.
    
- Make programs easier to maintain.
    
- Indicate that a value is fixed.
    

---

## Naming Convention

Constants are commonly written in uppercase letters.

```cpp
const double PI = 3.14159;
const int MAX_USERS = 100;
```

> [!note]  
> Uppercase naming is a convention, not a requirement.

---

## Constant Expressions

A constant expression is evaluated at compile time.

```cpp
const int SIZE = 10;
int arr[SIZE];
```

---

## `constexpr` (Modern C++)

Introduced in C++11.

```cpp
constexpr int SIZE = 100;
```

- Evaluated at compile time.
    
- Faster and more optimized than `const` in some cases.
    

### Example

```cpp
constexpr double PI = 3.1415926535;
```

> [!important]  
> Prefer `constexpr` when the value is known at compile time.

---

## Difference Between `const` and `constexpr`

|Feature|`const`|`constexpr`|
|---|---|---|
|Introduced|C++98|C++11|
|Compile-Time Guarantee|No|Yes|
|Runtime Values Allowed|Yes|No|

---

## Constant References

```cpp
int x = 10;

const int& ref = x;
```

- Prevents modification through the reference.
    
- Commonly used for function parameters.
    

---

## Example Program

```cpp
#include <iostream>

int main() {
    const int AGE = 21;

    std::cout << AGE;

    return 0;
}
```

---

## Important Points

- `const` makes a variable read-only.
    
- `constexpr` guarantees compile-time evaluation.
    
- Constants improve safety and maintainability.
    
- Use uppercase names for better readability.
    

---

## Common Mistakes

- Trying to modify a constant.
    
- Confusing `const` with `constexpr`.
    
- Using lowercase names for important constants.
    
- Assuming all `const` values are compile-time constants.
