| #   | Mistake                                                     | Reality                                                                                                                        | Advice                                                                                                                                                            |
| --- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | **Not Understanding the Asynchronous Nature of JavaScript** | JavaScript runs tasks asynchronously (e.g., `setTimeout`, Promises). Code does **not** always execute line‑by‑line.            | Study the difference between synchronous and asynchronous code early. Practice with callbacks, Promises, and `async/await` to internalize execution order.        |
| 2   | **Relying on JavaScript’s “Forgiving” Nature**              | Missing semicolons or sloppy variable declarations often don’t crash the program, but they create hidden bugs and messy logic. | Use `let`/`const` instead of `var`. Declare variables at the top of their scope. Enable linting (e.g., ESLint) to enforce strict syntax.                          |
| 3   | **Learning Outdated JavaScript**                            | Old tutorials omit modern ES6+ features like `Array.map`, `Array.filter`, `Array.reduce`, and strict equality (`===`).         | Choose up‑to‑date resources (2022+). Familiarize yourself with ES6+ syntax, modules, and new built‑in methods.                                                    |
| 4   | **Not Making Enough Projects**                              | Watching tutorials without building original work leads to shallow understanding.                                              | Build real‑world projects. Add unique value (e.g., turn a Tic‑Tac‑Toe game into a multiplayer app with WebSockets). Aim for deployable, interactive applications. |
| 5   | **Code Pollution (Messy Codebase)**                         | Unused functions and excessive globals make maintenance painful.                                                               | Keep the global scope clean. Remove dead code promptly. Adopt a framework (React, Next.js) for better organization as projects grow.                              |
