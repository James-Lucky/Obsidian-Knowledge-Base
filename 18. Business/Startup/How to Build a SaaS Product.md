# Phase 1 — Problem Discovery

## Goal

Find a painful problem that people already have.

### Sources

- Reddit
- X (Twitter)
- Indie Hackers
- Discord communities
- Facebook Groups
- App Store Reviews
- G2 Reviews
- Capterra Reviews

### Research Questions

Who has the problem?

What are they trying to accomplish?

Why are existing tools failing?

How much does the problem hurt?

Are they paying for workarounds?

---

# Output

```
User:FreelancersProblem:Missing client follow-ups.Current Solution:Generic todo apps.Pain:Lost payments and missed deadlines.
```

---

# Phase 2 — Validation

Before writing code, answer:

### Is the problem frequent?

### Is it painful?

### Are people actively searching for solutions?

### Will someone pay?

---

# Validation Scorecard

|Question|Yes/No|
|---|---|
|Happens often||
|Causes money loss||
|Existing tools bad||
|Users complain online||
|People pay for alternatives||

If fewer than 3 answers are "Yes", move on.

---

# Phase 3 — Problem Statement

Bad:

```
People need a better todo app.
```

Good:

```
Freelancers struggle to track client-related tasks becausegeneric task managers don't organize work around clients,causing missed follow-ups and delayed payments.
```

# Phase 4 — Solution Framing

Define one sentence:

```
This product helps [user]do [job]without [pain].
```

Example:

```
This product helps freelancers manage client tasks withoutmissing follow-ups.
```

# Phase 8 — MVP Scope

This is where most developers fail.

---

## MVP Rule

If removing a feature does not break the core job:

```
Remove it.
```

---

## Example

Core:

- Authentication
- Dashboard
- Create task
- Track task

Not MVP:

- AI
- Analytics
- Teams
- Notifications
- Themes
- Integrations

# Personal advice for you specifically

You often jump straight into building because you enjoy coding and ideas. That's fine for learning projects, but for a real SaaS business, reverse the order:

```
Research→ Validate→ Design→ Build→ Launch→ Iterate
```

not

```
Idea→ Build for 6 months→ Nobody uses it.
```
