# MapReduce Fundamentals

MapReduce is a programming model used for processing and analyzing massive amounts of data in distributed computing environments. It was developed by Google and later became the foundation of Big Data frameworks like Apache Hadoop.

Traditional systems cannot efficiently process terabytes or petabytes of data on a single machine. MapReduce solves this problem by dividing large tasks into smaller tasks and processing them in parallel across multiple computers.

The name “MapReduce” comes from two major phases:

1. Map Function
2. Reduce Function

MapReduce works on distributed systems and is highly scalable, fault-tolerant, and efficient for Big Data processing.

---

# 1. Understanding the Map Function

The **Map Function** is the first phase of MapReduce. Its main job is to process input data and convert it into intermediate key-value pairs.

The map function:

- Reads input data
- Divides data into smaller blocks
- Processes each block independently
- Produces intermediate output

Each mapper works in parallel on different data chunks.

---

# Working of the Map Function

Input data is split into small parts called **Input Splits**. Each split is processed by a mapper.

The mapper converts raw data into:

```
(Key, Value)
```

format.

---

# Example: Word Count Problem

Suppose the input text is:

```
Big Data is Big
```

The mapper reads each word and generates:

```
(Big, 1)(Data, 1)(is, 1)(Big, 1)
```

Each word becomes a key, and “1” becomes the value.

---

# Characteristics of Map Function

- Works independently
- Parallel processing
- Converts raw data into structured intermediate data
- Improves processing speed

---

# Importance

The map phase distributes workload across multiple machines, making Big Data processing faster and scalable.

---

# 2. Adding the Reduce Function

The **Reduce Function** is the second phase of MapReduce. It processes the intermediate key-value pairs generated by the map phase.

The reducer:

- Groups similar keys together
- Performs aggregation or computation
- Produces final output

---

# Working of Reduce Function

After mapping:

```
(Big, 1)(Data, 1)(is, 1)(Big, 1)
```

The framework groups similar keys:

```
(Big, [1,1])(Data, [1])(is, [1])
```

Reducer adds the values:

```
(Big, 2)(Data, 1)(is, 1)
```

---

# Functions Performed by Reducer

- Summation
- Counting
- Filtering
- Sorting
- Aggregation

---

# Example

An e-commerce company may use reducers to calculate:

- Total sales
- Product counts
- User activity frequency

---

# Advantages of Reduce Function

- Combines distributed results
- Produces summarized output
- Supports large-scale analytics

---

# 3. Anatomy of a MapReduce Job Run

A MapReduce job follows several steps from input to final output.

---

# Step-by-Step Execution

## Step 1: Input Data Storage

Data is stored in distributed storage systems like:

Hadoop Distributed File System.

Large files are divided into blocks.

---

## Step 2: Input Splitting

Input data is divided into smaller chunks called input splits.

Each split is assigned to a mapper.

---

## Step 3: Map Phase

Mappers process data and generate intermediate key-value pairs.

---

## Step 4: Shuffle and Sort

Intermediate data is:

- Grouped by key
- Sorted automatically

This phase transfers mapper output to reducers.

---

## Step 5: Reduce Phase

Reducers process grouped data and generate final output.

---

## Step 6: Output Storage

Final results are stored back into HDFS.

---

# Example Flow

Input:

```
Apple Banana Apple
```

Map Output:

```
(Apple,1)(Banana,1)(Apple,1)
```

Shuffle and Sort:

```
Apple → [1,1]Banana → [1]
```

Reduce Output:

```
(Apple,2)(Banana,1)
```

---

# 4. Failures in MapReduce

Since MapReduce runs on distributed systems with many machines, failures are common.

Failures may occur due to:

- Hardware crashes
- Network issues
- Disk failures
- Power failures

---

# Fault Tolerance in MapReduce

MapReduce automatically handles failures.

If a node fails:

- Tasks are reassigned to another node
- Replicated data is used
- Processing continues

---

# Types of Failures

## Task Failure

A single map or reduce task crashes.

Solution:  
Task is restarted automatically.

---

## Node Failure

Entire machine stops working.

Solution:  
Tasks move to another machine.

---

## Network Failure

Communication between nodes fails.

Solution:  
System retries communication.

---

# Example

If one mapper crashes while processing data, Hadoop reruns the same mapper on another node using replicated data from HDFS.

---

# Importance

Fault tolerance is one of the biggest strengths of MapReduce because Big Data clusters may contain thousands of machines.

---

# 5. Job Scheduling

Job scheduling determines:

- Which task runs
- On which machine
- At what time

Efficient scheduling improves performance and resource utilization.

---

# Types of Scheduling

## FIFO Scheduling (First In First Out)

Jobs execute in order of submission.

Simple but inefficient for mixed workloads.

---

## Fair Scheduling

Resources are distributed fairly among users and jobs.

---

## Capacity Scheduling

Cluster resources are divided into queues with fixed capacities.

Used in large organizations.

---

# Example

If multiple users submit jobs simultaneously, the scheduler decides how CPU and memory resources are allocated.

---

# Importance

Good scheduling:

- Reduces waiting time
- Balances workload
- Improves cluster efficiency

---

# 6. Shuffle and Sort

Shuffle and Sort is the intermediate phase between map and reduce.

It is one of the most important parts of MapReduce.

---

# Shuffle Phase

The system transfers mapper output to reducers.

Data with the same key is sent to the same reducer.

---

# Sort Phase

Keys are automatically sorted before reaching reducers.

---

# Example

Mapper Output:

```
(Cat,1)(Dog,1)(Cat,1)
```

After Shuffle and Sort:

```
Cat → [1,1]Dog → [1]
```

---

# Importance

Shuffle and sort:

- Organize data efficiently
- Enable accurate reduction
- Improve processing performance

---

Apache Hadoop is an open-source framework used for storing and processing massive amounts of data across distributed computer systems. It was developed to handle Big Data efficiently using clusters of low-cost hardware instead of expensive supercomputers.

Traditional database systems struggle when dealing with:

- Petabytes of data
- Unstructured information
- High-speed processing requirements

Hadoop solves these problems using:

- Distributed storage
- Parallel processing
- Fault tolerance
- Scalability

It became one of the most important technologies in Big Data processing.

---

# HDFS (Hadoop Distributed File System)

Hadoop Distributed File System (HDFS) is the storage layer of Hadoop designed to store and manage very large datasets across multiple machines in a distributed environment.

Traditional file systems cannot efficiently handle:

- Terabytes or petabytes of data
- Distributed storage
- Hardware failures
- Parallel processing requirements

HDFS solves these problems using:

- Distributed storage
- Fault tolerance
- Data replication
- High scalability

It is one of the core components of Apache Hadoop.

---

# 1. The Design of HDFS

HDFS is designed based on the idea that:

- Hardware failures are common
- Large files must be processed efficiently
- Data should be distributed across many machines

The design focuses on:

- High throughput
- Reliability
- Scalability
- Fault tolerance

---

# Main Design Goals of HDFS

## A. Distributed Storage

Data is stored across multiple machines called nodes.

---

## B. Fault Tolerance

Data is replicated automatically so it can survive node failures.

---

## C. High Throughput

Optimized for processing huge datasets quickly.

---

## D. Scalability

New nodes can be added easily to increase storage and performance.

---

# Example

A 1 TB video dataset is divided into smaller blocks and distributed across many servers.

---

# HDFS Architecture

HDFS follows a **Master-Slave Architecture**.

---

# Main Components

## A. NameNode (Master Node)

The NameNode manages metadata.

It stores:

- File names
- Block locations
- Permissions
- Directory structure

The NameNode does not store actual file data.

---

## B. DataNode (Slave Node)

DataNodes store actual file blocks.

Functions:

- Read/write data
- Send block reports
- Replicate blocks

---

# Example

Suppose a file is divided into 3 blocks:

- Block A → Node 1
- Block B → Node 2
- Block C → Node 3

The NameNode maintains information about all block locations.

---

# 2. HDFS Concepts

Several important concepts are used in HDFS.

---

# A. Blocks

Files are divided into fixed-size blocks.

Default block size:

- 128 MB (commonly)

---

# Why Large Block Size?

Large blocks:

- Reduce metadata overhead
- Improve processing efficiency

---

# Example

A 500 MB file may be divided into:

- 128 MB
- 128 MB
- 128 MB
- 116 MB

---

# B. Replication

Each block is copied multiple times.

Default replication factor:

```
3
```

This means:

- One original block
- Two backup copies

---

# Importance

If one node fails, another copy remains available.

---

# C. Rack Awareness

HDFS stores replicas on different racks to improve fault tolerance.

---

# D. Streaming Data Access

HDFS is optimized for:

- Large sequential reads
- Batch processing

Not suitable for:

- Low-latency random access

---

# E. Write Once Read Many

Files are generally written once and read multiple times.

---
