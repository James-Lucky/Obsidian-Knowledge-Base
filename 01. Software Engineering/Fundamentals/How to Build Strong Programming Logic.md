## 1. Focus on Concepts  
  
> **Concepts > Contests**  
  
- Emphasize understanding concepts before competitive programming.  
  
---  
  
## 2. Thinking Before Coding  
  
Before writing code:  
  
- [ ] Read the question carefully.  
- [ ] Use pen and paper.  
- [ ] Think about corner cases.  
- [ ] Dry run at least **2 sample inputs**.  
  
---  
  
## 3. Solve Step-by-Step  
  
### Step 1: Brute Force  
  
- Solve the problem first using the simplest approach.  
- Don't worry about efficiency initially.  
  
### Step 2: Optimize  
  
- Reduce the time complexity.  
- Improve the space complexity if possible.  
- Look for better algorithms or data structures.  
  
---  
  
## Common Optimization Techniques  
  
- Two Pointers  
- Sliding Window  
- Prefix Sum  
- Hashing  
- Binary Search  
- Greedy  
- Recursion  
- Backtracking  
- Dynamic Programming  
- Graph Algorithms  
- Trees  
- Heap / Priority Queue  
  
---  
  
## 4. Practice Daily  
  
- [ ] Solve problems every day.  
- [ ] Consistency matters more than solving many problems in one day.  
  
---  
  
## 5. Move to New Problems  
  
Assume interview expectations:  
  
| Difficulty | Expected Time |  
|------------|---------------|  
| Easy | 5–10 min |  
| Medium | 15–20 min |  
| Hard | 40–60 min |  
  
If you cannot solve within the expected time:  
  
- Review the solution.  
- Understand the logic.  
- Rewrite it without looking.  
- Revisit the problem after a few days.  
  
---  
  
## 6. Solve Coding Puzzles  
  
Coding puzzles improve:  
  
- Logical thinking  
- Pattern recognition  
- Problem-solving speed  
- Creativity
